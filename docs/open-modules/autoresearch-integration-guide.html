🧭 Framework Placement

  • Associated Workflows: Merged directly as a sub-profile under I-Wish canonical /autoresearch skill.
  • Associated Agents: dev-agent (code generator/hyperparameter tuner) and review-agent (safety and metrics analyzer).
  • Workspace promotion: Python math utilities (Muon, Best-fit loader) compiled to user workspace _iwish-output/libs/.

⚡ Key Decisions

  • NO Loop-Forever: Rejected the infinite prose loop in favor of I-Wish's structured 8-phase state-machine with fixed 10-iteration budget bounds.
  • NO Destructive Resets: Banned raw git reset --hard commands on user workspace. Replaced with safe rollback hooks.
  • Vulnerability Patching: Audit uncovered 10 vulnerabilities in secondary Python packages (Pillow, requests, idna). Mitigated by forcing pinned upgrades.

🔁 Input ➔ Process ➔ Output

1. Setup & Preconditions
Validates PyTorch CUDA environment, ensures git working tree is clean, and reads target hyperparameters.
2. Modify & Commit
The optimization agent modifies model structure or optimizer weights in train.py and commits changes.
3. Verify (Time-Budget Run)
Runs a timed (5 minutes) validation run. Extracts final val_bpb (Bits-per-byte) metric and peak VRAM.
4. Ratchet Decision
Keeps the changes if validation loss improves. Performs safe rollback if metric degrades or training crashes. Logs to loop-results.tsv.

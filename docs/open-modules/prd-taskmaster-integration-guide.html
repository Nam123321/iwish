prd-taskmaster Integration

Tích hợp bộ lọc từ khóa mơ hồ và mô hình cooldown-throttled cập nhật vào hệ thống I-Wish

🧭 Kế hoạch Tích hợp & Đóng gói

Dựa trên phê duyệt của người dùng đối với hướng tiếp cận Selective Pattern Adoption, chúng ta sẽ trích xuất hai mẫu thiết kế tinh hoa từ prd-taskmaster và tích hợp trực tiếp vào hệ thống I-Wish.

Phương án: Đóng gói các tính năng dưới dạng System Skill (SYSTEM_SKILL), nâng cấp quy trình validation spec tự động mà không bị phụ thuộc vào môi trường Taskmaster bên ngoài.

1. Bộ lọc từ mơ hồ (Vague Language Filter)

Tích hợp bộ regex phân tích và phát hiện các cụm từ định lượng mơ hồ (e.g. "nhanh", "an toàn", "fast", "scalable") trong văn bản PRD, buộc tác nhân AI phải ghi nhận các chỉ số đo lường (SMART metrics) cụ thể.

2. Cơ chế Cooldown API (Throttled Update Checker)

Áp dụng helper function sử dụng cache thời gian nhằm giới hạn số lần gọi các cổng API ngoài (HuggingFace, GitHub Releases, các dịch vụ ontology) để tránh bị khóa rate-limit trong các skill lập trình và nghiên cứu.

🛡️ Biên giới & Ràng buộc (Boundaries & Constraints)

  • Vocabulary Scope: Một số cụm từ kiểm tra có thể trùng lặp với từ khóa hợp lệ trong phát biểu kỹ thuật (ví dụ: "secure SSL connection"). Logic regex sẽ cần lọc kỹ để tránh cảnh báo sai.
  • Sandbox Isolation: Tất cả cache và file config của bộ đệm cooldown phải ghi vào thư mục brain an toàn của I-Wish (~/.gemini/antigravity/brain/), tuyệt đối không ghi ra ngoài workspace để giữ cô lập bảo mật.
  • Performance: Đối với tài liệu spec cực lớn (>10,000 dòng), regex có thể chạy chậm. I-Wish sẽ tự động phân khúc văn bản theo heading để quét tuần tự nhằm tối ưu bộ nhớ.

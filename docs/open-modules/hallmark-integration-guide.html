2. Why it exists

Problem 1: Fragmented & Fragile Cloners. Standard website cloning utilities like clone-website attempt to duplicate DOM structures and raw styles synchronously. This breaks easily under Single Page Applications (SPAs), introduces heavy inline styles, and lacks responsive structural adaptation.

Problem 2: The "AI Design" Tell. AI-generated layouts are notoriously sloppy. They frequently suffer from generic typeface pairings (e.g., standard Inter/Roboto combos), default linear gradients (purple-to-blue), lazy transition timing (transition-all on everything), lack of state feedback (no active, focus, or disabled states), and uneven margins.

Solution: Hallmark introduces Design DNA Study Mode, which extracts layout skeleton rules, colors, and rhythm without duplicating pixel-for-pixel code. It then builds code utilizing clean, modern tokens, and verifies the layout against a 69-Gate Slop Test and enforces an **8-State Interactive Preview**.

5. The 5-Step DNA Study Protocol

When analyzing a reference site, Hallmark follows a strict, sequential pipeline to decode the visual skeleton:

1

Surface (Color & Vibe)

Analyzes paper lightness bands (dark/light/mid), background color temperature, anchor accent hue, and accent footprint percentage.

2

Type (Typography Roles)

Maps typefaces to roles: Display role (e.g., italic editorial serif), Body role (e.g., neutral grotesque), and Label role (e.g., monospace). Records exact names in URL mode.

3

Structure (Layout Archetypes)

Identifies layout skeletons (e.g., Bento Grid, Split Studio, Hero Specimen) and component layout knobs.

4

Motion (Keyframes & Interactions)

Reads libraries loaded (GSAP, Lenis, Framer Motion) and custom `@keyframes` variables.

5

Rhythm (Spacing & Density)

Gauges visual tension, negative space, and padding scale. (Marked as a blind spot in URL Mode).

6. Constraints & Edge Cases

URL Mode Rhythm Blind Spot: WebFetch parses HTML source code, meaning it cannot visualize actual layout margins and padding density. Rhythm defaults to unknown (URL mode). Attach a screenshot alongside the URL if spacing analysis is required.

Empty SPA Mounts: Pages rendered entirely client-side (e.g., standard Create React App, Webflow redirects) fail on fetch as only the initial script mount tag is returned. Hallmark fallbacks to screenshot image input.

Context Window Tax: Hallmark's full design prompts are extremely large (~350KB total). Injected dynamically into every prompt, they cause slow processing. We mitigate this by extracting key rules and adding them to targeted UX Guardian profiles.

7. Example: 8-State Interactive CSS Pattern

Hallmark mandates that every interactive element defines styling for all eight key interactive states to ensure a premium UX. Here is an example of an integrated CSS component:

/* Button - 8 States Enforced */
.btn-action {
  /* 1. Default State */
  background-color: oklch(0.65 0.2 260);
  color: #ffffff;
  border-radius: 8px;
  transition: background-color 150ms cubic-bezier(0.16, 1, 0.3, 1);
}

.btn-action:hover {
  /* 2. Hover State */
  background-color: oklch(0.58 0.2 260);
}

.btn-action:active {
  /* 3. Active State */
  background-color: oklch(0.5 0.2 260);
}

.btn-action:focus-visible {
  /* 4. Focus State */
  outline: 2px solid oklch(0.65 0.2 260);
  outline-offset: 3px;
}

.btn-action:disabled {
  /* 5. Disabled State */
  opacity: 0.45;
  cursor: not-allowed;
}

/* Modifier classes for advanced states */
.btn-action.is-loading {
  /* 6. Loading State */
  cursor: wait;
}

.btn-action.is-selected {
  /* 7. Selected State */
  box-shadow: 0 0 0 2px #ffffff;
}

.btn-action.has-error {
  /* 8. Error State */
  background-color: var(--danger);
}

8. User Review Questions

  1. Do you want the 8-state component preview generation (via .preview.html) to be automatically run for all frontend tickets, or only when explicitly requested?
  2. How strictly should we block builds that fail our visual anti-slop checks (e.g. use of pure black/white background colors, default transition-all definitions)?
  3. For reference site imports, would you prefer using screenshots by default to ensure Rhythm/negative space analysis is always included?

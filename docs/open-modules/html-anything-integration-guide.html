I-Wish Open Module Guide

HTML Anything trong I-Wish

`html-anything` là một external capability bundle rất mạnh để biến output của I-Wish thành artifact HTML đọc tốt, trình bày tốt, share tốt. Trong hệ của chúng ta, nó không phải xương sống delivery mà là một supportive compound module phục vụ nhiều phase từ idea đến deliver.

Kết luận ngắn: `html-anything` không thay thế `plan`, `make-ui-spec`, `review`, `code` hay `research`. Nó tăng chất lượng output theo hướng human-facing: dễ đọc, dễ chụp màn hình, dễ share, dễ handoff.

Vị trí trong khung phát triển

Trong I-Wish, user nên nhìn hệ thống theo trục Delivery Framework → Process-primary capability → Supportive capability → Foundational injection. `html-anything` nằm ở lớp supportive capability.

Discover

Research artifact

Tóm tắt market/domain/technical research thành HTML brief hoặc deck dễ đọc.

Primary flows: research, bmad-bmm-market-research

Plan

Product brief / PRD packaging

Biến brief, PRD summary, roadmap thành output có thể share với stakeholder.

Primary flows: plan, bmad-bmm-create-prd

Solution

UI/UX support

Tạo UI concept page, docs page, prototype HTML, visual review artifact.

Primary flows: make-ui-spec, bmad-bmm-create-ui-spec

Implement

Implementation handoff

Tạo implementation note, bug report, review artifact hoặc prototype gần với thực thi.

Primary flows: code, fix-bug, review

Validate / Deliver

Status & communication

Tạo sprint report, release note, status deck, stakeholder-facing summary.

Primary flows: status, retro

Operate / Learn

Knowledge packaging

Đóng gói learning, guide, repo summary, reusable docs sau absorb hoặc evolution.

Primary flows: absorb-repo, create-skill, enhance-skill

Nguyên tắc dùng: Orch không nên route trực tiếp vào `html-anything` như main flow. Nó nên route vào process chính trước, rồi gắn `html-anything` khi user cần output dạng docs/deck/prototype/report/shareable HTML.

Tình huống sử dụng thực tế

Case 1 — Product brief dễ share

User: “Tạo product brief HTML để share với stakeholder.”

Orch: pm-agent + /plan + supportive module `html-anything`

Output gợi ý: `pm-spec`, `doc`, `deck-pitch`, `deck-simple`

Case 2 — UX lỗi so với master design

User: “UX có nhiều lỗi chưa tuân thủ master design.”

Orch: ux-agent + /make-ui-spec hoặc /review

Supportive: `design-consultation`, `ux-guardian`, `ui-ux`, `html-anything`

Case 3 — Sprint status deck

User: “Tạo deck sprint status để họp với stakeholder.”

Orch: delivery-manager-agent hoặc pm-agent + /status

Output gợi ý: `weekly-update`, `deck-tech-sharing`, `deck-simple`

Case 4 — Prototype HTML trước khi code

User: “Tạo prototype HTML cho landing page để team góp ý trước khi code.”

Orch: ux-agent + /make-ui-spec + `html-anything`

Output gợi ý: `prototype-web`, `saas-landing`, `docs-page`, `dashboard`

Khi không nên dùng: backend-only work, DB-only work, governance decisions, source-of-truth PRD/story ownership, hoặc task không cần human-facing artifact.

Tài liệu này được tạo như một onboarding artifact cho Open Module test của I-Wish với nexu-io/html-anything.

SYSTEM_SKILL Selective Extraction Bilingual

1. Tổng quan & Phân loại / What is it

  • Tên thư viện / Repo name: Ruflo (trước đây là Claude Flow)
  • Nguồn / Source: https://github.com/ruvnet/ruflo
  • Trạng thái đăng ký / Registration state: Phê duyệt trích xuất chọn lọc làm SYSTEM_SKILL.
  • Phân loại Hình dạng / Shape: compound (tệp phức hợp chứa các module con).
  • Phân loại Vai trò / Role: supportive (hỗ trợ quy trình tối ưu hóa và chống lỗi).

2. Lý do Tích hợp / Why it exists

Tích hợp Ruflo nhằm giải quyết ba lỗ hổng hiệu năng và quy trình chính trong I-Wish:

  • Bypass LLM: Giảm thời gian chờ và chi phí gọi API đối với các chỉnh sửa định dạng mã nguồn đơn giản.
  • Tự động phục hồi: Giảm công sức dọn dẹp thủ công bằng cách rollback tự động các tệp tin sửa đổi nếu Story bị compile error giữa chừng.
  • Định tuyến linh hoạt: Cân đối hiệu năng và hóa đơn API LLM dựa trên học máy tăng cường thay vì cấu hình tĩnh.

3. Vị trí trong Khung phân phối / Framework Placement

  • Phần chặng phục vụ / Phases: Nằm ở quy trình phát triển mã /code, quy trình Story /iwish-feature-dev-story và định tuyến gọi mô hình ở orch-agent.
  • Vai trò / Role: supportive. Hoạt động ngầm làm tấm chắn bảo vệ và tối ưu hóa trước khi LLM chính xử lý tác vụ.

4. Mô hình Dữ liệu: Vào -> Xử lý -> Ra / Input -> Process -> Output

Agent Booster (LLM Bypass)

Input: Mã nguồn đích + Ý định sửa đổi.

Process: So khớp các mẫu biểu thức chính quy (regex) cục bộ qua WASM.

Output: File đã sửa đổi (nếu khớp regex) hoặc chuyển giao cho LLM.

Topological Rollback

Input: Danh sách sub-tasks đã chạy thành công + Các hook rollback tương ứng.

Process: Khi bước chạy test/build thất bại, gọi hàm rollback theo thứ tự đảo ngược.

Output: Hoàn trả mã nguồn về trạng thái commit sạch ban đầu.

5. Các Kịch bản Sử dụng / Use Cases

Kịch bản Cốt lõi / Core Use Cases

  • Formatting & Trivial Edits: Tự động bọc try-catch, định dạng mã, JSDoc comments mà không tốn token.
  • Auto-Cleanup on Test Failure: Revert các file nguồn về commit gần nhất khi Story chạy test lỗi.
  • Adaptive LLM Routing: Phân phối task nhẹ sang model rẻ và task nặng sang model cao cấp.

Trường hợp Cấm sử dụng / Do-Not-Use Cases

1. Codebase Search: Tuyệt đối KHÔNG dùng HNSW/SQLite WAL của Ruflo. I-Wish đã tích hợp thành công Semble (giảm 98% token tìm kiếm thông qua /code-search) với hiệu năng vượt trội.

2. Setup Scripts: Tuyệt đối CẤM chạy các file cài đặt install.sh hoặc sync-skills.sh của Ruflo để bảo vệ an toàn Project Isolation.

6. Các Trường hợp Biên & Hạn chế / Edge Cases & Constraints

Regex CPU Spike (ReDoS): Có thể bị treo CPU khi chạy regex phức tạp.

Biện pháp: Cài đặt timeout tối đa 100ms cho Agent Booster.

Thompson Router Vendor Lock-in: Nguyên bản của Ruflo chỉ hỗ trợ Claude.

Biện pháp: Thiết kế mở rộng động để lấy danh sách các model khả dụng từ biến môi trường của I-Wish.

7. Phối hợp & Chỉ dẫn Định tuyến / Coordination & Routing Hints

Từ khóa kích hoạt (Trigger words): định dạng mã, thêm try-catch, rollback tác vụ, định tuyến model.

Từ khóa chặn (Anti-triggers): tìm kiếm code (chuyển hướng sang Semble), cài đặt ruflo (bỏ qua).

Được đề xuất tự động (auto-proposed) bởi Trình điều phối chính dưới dạng một lớp đệm tối ưu hóa trong luồng chạy.

1. What is it

The agent-skills repository is a curated collection of production-grade engineering guidelines, workflows, and local execution hooks. Instead of treating it as a single monolithic block, I-Wish adopts it through Selective Extraction. Its architectural elements are mapped into specialized, lightweight fragments (which inject custom instructions directly into workflows) and independent skills (which package functional scripts).

2. Why it exists

AI agents naturally suffer from overconfidence, context-flooding, and "laziness" (e.g. leaving placeholders or avoiding rigorous validation). This integration introduces:

  • Doubt-Driven Development (DDD): A system where the agent is forced to mentally test and disprove its own claims using a zero-logic sandbox.
  • SDD Cache (Prompt Caching): An extremely secure, revalidating local cache layer for fetch tools that checks HTTP ETags/Last-Modified headers, saving massive token budgets.
  • Code Simplification: Strict metrics targeting nesting levels and function lengths to ensure code readability isn't sacrificed for brevity.

6. Edge Cases, Risks & Constraints

SSRF Redirect Egress (PR #294 Vulnerability Fix)

The original cache shell pre-hook allowed curl to follow arbitrary redirects, opening up SSRF vectors on loopbacks or metadata services (e.g. 169.254.169.254).

Remediation: Force --max-redirs 0 and validate target URLs against a local subnet whitelist before execution.

Cache Symlink Traversal

Unvalidated cache file writing could follow malicious symlinks placed inside the cache folder, leading to arbitrary host file overwrites.

Remediation: Pre-verify that the target cache path is a regular file or does not exist, and restrict cache files permissions to 0600.

Doubt Loop Prevention

Adversarial agents can fall into infinite self-skepticism loops, wasting substantial context tokens.

Remediation: Set a strict limit of three (3) doubt iterations before forcing escalation to the human user.

7. Swarm Coordination

Different specialized agents inside the I-Wish swarm leverage these skills coordinate:

  • review-agent: Acts as the adversary for Doubt-Driven Development reviews. Strips target intent during checks.
  • dev-agent: Integrates Code Simplification metrics and executes incremental TDD checks.
  • research-agent: Orchestrates the Source-Driven doc fetching and citation formatting.
  • orch-agent: Evaluates debate outcomes and routes unresolved conflicts to the user.

8. Orch Router Hints

The orchestrator routes requests containing specific cues:

  • Trigger Keywords: revalidate cache, verify complexity, doubt check, reduce nesting, LCP, ADR.
  • Anti-Triggers: rename, typo, format.

11. Codebase Health & Code Simplification Synergy

To ensure high-quality codebase maintenance without conflict:

The Feedback Loop: Diagnosis to Action

Codebase Health (/codebase-health) is a Macro Diagnostic Gate that scans the entire repository statically to detect files and functions that violate architectural boundaries or exceed metric thresholds (e.g. connections > 50, nesting >= 3).

Code Simplification (code-simplification) is a Micro Remediation Skill. It operates during active coding (in-flight) to give the developer agent step-by-step instructions on how to rewrite complex parts (using early returns, guard clauses, splitting files) to resolve the hotspot.

Coexistence Rules

  1. Targeted Hotspots: Files flagged by /codebase-health during milestone reviews are automatically marked for simplification.
  2. In-flight Enforcement: If a developer agent is editing a file and either notices it is flagged as a health hotspot, or if the agent's new edits would push it beyond complexity thresholds, the agent must apply the code-simplification rules to simplify the module before completing the task.
  3. Behavioral Parity: Every simplification step must be immediately tested to guarantee that behavior is exactly preserved.

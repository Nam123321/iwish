Harness Integration Guide

1. Classification & Framework Placement

Shape: workflow skill-attachment

Role: process-primary supportive

Framework Placement: SYSTEM_SKILL (Direct integration into I-Wish system workflows such as /create-architecture, /create-skill, /tournament, and qa-agent).

2. Core Use Cases

  • Architecting Multi-Agent Teams: Leveraging the 6-Pattern Architecture matrix to define exact multi-agent topologies.
  • Skill Generation: Adopting Progressive Disclosure (3-layer: Metadata -> Main -> References) to ensure skills remain under context limits.
  • A/B Skill Testing: Injecting "With vs Without" testing into /tournament to definitively validate skill effectiveness.

3. Adjacent Use Cases

  • Incremental QA Validation: Upgrading the qa-agent guide to perform boundary cross-checks incrementally during development.

4. Edge Cases

  • Compound Skill Explosion: Managing massive reference structures for highly complex skills.
  • Tournament Token Limits: Handling rate limits during parallel "With vs Without" agent executions.

5. Stress Cases

  • Attempting to architect a 10+ agent team. I-Wish will adopt Harness's hard limit advising 3-5 focused agents.

6. Constraints

  • Do not adopt the CLAUDE.md pointer system (I-Wish uses project-context.md).
  • Do not blindly force Agent Teams for simple tasks.

7. Agent / Workflow / Skill Coordination

  • architect-agent: Consumes the 6-Pattern Matrix.
  • capability-agent: Uses the 3-Layer Prompt Pattern and "With vs Without" testing.
  • qa-agent: Implements Incremental Boundary Checks.

Orch should explicitly append hints for progressive disclosure when routing to /create-skill.

# Idea Challenge Integration Guide

## What It Is

`idea-challenge` is I-Wish's canonical Discover-phase workflow for challenging a product or solution concept before heavy planning begins.

It absorbs the useful spine of upstream PRFAQ / Working Backwards, but is adapted for I-Wish orchestration.

## What It Helps With

- validating whether an idea is clear enough to plan
- surfacing customer objections early
- checking whether a concept is compelling before PRD
- preparing a cleaner handoff into `/plan`

## Stage Flow

- `step-ic-01-ignition`
- `step-ic-02-press-release`
- `step-ic-03-customer-faq`
- `step-ic-04-internal-faq`
- `step-ic-05-strategic-advantage`
- `step-ic-06-distillate-and-handoff`

## Related Capabilities

- owner workflow: `idea-challenge`
- questioning add-on: `socratic-review`
- strategic add-on: `unique-advantage-evaluator`
- evidence add-on: `research`

## Input -> Process -> Output

### Input

- an idea, concept, or product direction
- optional existing brief, notes, or research

### Process

- clarify problem/user/stakes/solution
- challenge claims through customer and stakeholder lenses
- attach strategic or research support when needed

### Output

- planning-ready concept artifact
- optional distillate
- optional `biz-stack`

## Use Cases

- new product idea
- product concept still feels fuzzy
- before PRD
- after a major strategic pivot

## Constraints

- this is not a code workflow
- market or competitor claims should be grounded via `research`
- use `unique-advantage-evaluator` when differentiation matters

# SKILL: Brand Identity & Brand Guideline Workflow

## Purpose

Use this skill to create a professional Brand Identity and Brand Guideline system for any brand, especially SaaS, AI, technology, workspace, B2B, retail, e-commerce, edtech, manufacturing, or service businesses.

The workflow covers:
- Brand strategy
- Brand identity direction
- Logo system
- Color system
- Typography system
- Iconography system
- Visual language
- Product/UI applications
- Marketing applications
- Advertising creative systems
- Brand guideline documentation
- Asset folder structure

---

## Core Principle

A professional brand identity should not only look good. It must be:
1. Strategically aligned with the business.
2. Consistent across product, marketing, sales, and physical touchpoints.
3. Practical for daily use by designers, marketers, sales teams, product teams, and vendors.
4. Easy to extend into future campaigns, product modules, and new markets.
5. Documented clearly enough that another designer can continue the system.

---

## Recommended Workflow

### Phase 1 — Brand Discovery

Collect required inputs using the questionnaire.

Minimum required information:
- Brand name
- Industry
- Product/service description
- Target audience
- Brand positioning
- Brand personality
- Visual preferences
- Competitors / references
- Required deliverables
- Output format

Do not start final guideline production until the core strategic inputs are clear.

---

### Phase 2 — Brand Strategy Foundation

Create the following:

1. Brand overview
2. Mission
3. Vision
4. Positioning statement
5. Target audience
6. Use cases
7. Brand personality
8. Brand values
9. Brand voice and tone
10. Messaging hierarchy
11. Main tagline and alternative tagline options

Recommended output:
- 1 concise strategy page
- 1 messaging page
- 1 brand personality page

---

### Phase 3 — Visual Direction Exploration

Create 3–10 visual directions depending on project scope.

For each direction, define:
- Theme name
- Visual style
- Primary color
- Secondary color
- Neutral palette
- Typography mood
- Logo treatment
- UI/product implications
- Pros and cons
- Best use case
- Risk level

Evaluation criteria:
- Professional
- Trustworthy
- Differentiated
- Scalable
- Suitable for daily usage
- Suitable for product UI
- Suitable for marketing
- Suitable for target customers
- Easy to apply across channels

---

### Phase 4 — Logo System

Create and document:

1. Primary logo
2. Horizontal lockup
3. Vertical lockup
4. Symbol-only mark
5. Wordmark-only version
6. App icon
7. Favicon
8. Monochrome versions
9. Reversed versions
10. Light background versions
11. Dark background versions
12. Color background versions
13. Clearspace
14. Minimum size
15. Incorrect usage
16. Accessibility and contrast notes

Important:
- Logo assets should be exported as high-quality image files or SVG files.
- Avoid using transparent PNGs with noisy edges on dark backgrounds.
- For dark backgrounds, create separate logo assets optimized for dark mode.
- Do not rely on rough white-background removal when preparing production logo files.
- Keep naming consistent.

Recommended naming:
- brand-primary-light.svg
- brand-primary-dark.svg
- brand-symbol-light.svg
- brand-symbol-dark.svg
- brand-primary-mono-black.svg
- brand-primary-mono-white.svg
- brand-symbol-mono-black.svg
- brand-symbol-mono-white.svg
- brand-app-icon-light.png
- brand-app-icon-dark.png

## Quality Checklist

Before final delivery, check:

### Strategy
- Brand positioning is clear.
- Target audience is specific.
- Brand personality is consistent.
- Messaging is practical and not generic.

### Logo
- Primary logo is clear.
- Symbol works at small sizes.
- Dark mode logo is separately optimized.
- Monochrome logo works.
- Clearspace and minimum size are documented.
- Incorrect usage is shown.

### Color
- Light mode palette is usable.
- Dark mode palette is usable.
- Contrast is acceptable.
- Accent colors are not overused.
- Semantic colors are defined.

### Typography
- Type scale is coherent.
- Fonts match brand personality.
- Body copy is readable.
- UI labels are legible.
- Multilingual support is considered.

### UI / Product
- Dashboard looks realistic.
- Navigation structure is clear.
- Workflow/task/project states are represented.
- Human and AI roles are visually distinct.
- Components are reusable.

### Marketing
- Billboard has instant readability.
- Booth/POSM has clear touchpoints.
- Brochure has front/back hierarchy.
- Social ads have strong hooks.
- Website hero has clear CTA.
- Email design has conversion logic.

### Packaging
- Files are named consistently.
- Assets are organized.
- HTML works standalone or has correct relative paths.
- Logo images do not have noisy transparency edges.
- README explains how to use the package.

# STORY-NAV-1.3: Markdown Rendering Engine Integration

## 1. Business Context
As the I-Wish Idea Navigator evolves, the artifacts generated by AI agents (like PRDs, Architecture docs, and Epics) contain rich Markdown formatting. Currently, the navigator injects this content as plain text or unsafe HTML via `innerHTML`. To ensure a professional, secure, and visually stunning "Brainstorm-inspired" experience, we must integrate a robust Markdown parser and a strict HTML sanitizer. This allows us to display rich documents securely while applying our "Celestial Realm" design language to the parsed tags.

## 2. Tracer Bullet (Vertical Slice)
**Data Flow:** `window.I-Wish_NAV_DATA` -> `marked.js` parsing -> `DOMPurify` sanitization -> Safe injection into `<dialog>` element -> Styled by `navigator-theme.css`.
**Integrity:** Covers UI rendering, frontend business logic (parsing/sanitization), and secure DOM injection. No backend changes required.

## 3. Technical Requirements
- **Parser:** Integrate `marked.js` (via reliable CDN) to parse Markdown to HTML.
- **Sanitizer:** Integrate `DOMPurify` (via reliable CDN) to strip malicious scripts and XSS vectors.
- **Styling:** CSS rules targeting `.markdown-body` inside `navigator-theme.css` to ensure standard HTML tags (`h1`-`h6`, `p`, `ul`, `li`, `blockquote`, `pre`, `code`) inherit the Glassmorphism and Neon Mint aesthetic.
- **Performance:** Ensure synchronous parsing does not noticeably block the main thread for typical artifact sizes.

## 4. Acceptance Criteria (AC)
- **[AC1]** `marked.js` successfully parses standard Markdown elements (headings, lists, code blocks, bold/italic) from the artifact content into valid HTML.
- **[AC2]** `DOMPurify` actively intercepts the parsed HTML, removing any `<script>`, `onload` attributes, or other XSS vectors before DOM insertion.
- **[AC3]** The injected HTML renders beautifully inside the `<dialog>` modal, with styling (typography, spacing, glowing accents) dictated by `navigator-theme.css` without breaking the glassmorphism layout.
- **[AC4]** The implementation gracefully handles missing or empty content gracefully, and does not throw errors if the CDNs fail to load (fallback to raw text).

## 5. Implementation Tasks
- [ ] **Task 1: Library Integration:** Add `<script>` tags for `marked.js` and `DOMPurify` via CDN (e.g., jsDelivr) into the `<head>` of `idea-navigator.html`.
- [ ] **Task 2: Engine Refactoring:** Update `navigator-engine.js` `openModal()` function. If the artifact has `content`, pass it through `marked.parse()`, then `DOMPurify.sanitize()`, and inject the result into a `.markdown-body` container inside the modal.
- [ ] **Task 3: Celestial Styling:** Append `.markdown-body` specific CSS rules to `navigator-theme.css`. Style headers with Neon Mint, `blockquote` with glass backgrounds and left borders, and `pre code` blocks with deep navy/terminal aesthetics to maintain the "Brainstorm" spirit.

## 7. Dev Notes
- **CDN Links to use:**
  - Marked: `https://cdn.jsdelivr.net/npm/marked/marked.min.js`
  - DOMPurify: `https://cdnjs.cloudflare.com/ajax/libs/dompurify/3.0.6/purify.min.js`
- **Styling Note:** Do not use Tailwind for the markdown body. Write vanilla CSS rules nested under `.markdown-body` to keep parity with the project's design constraints.

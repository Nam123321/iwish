🚀 Trục Phát Triển Chính (Main Pipeline)

Quy trình phát triển phần mềm trong I-Wish được thiết kế theo mô hình AI-Native Factory gồm 6 giai đoạn rõ ràng từ ý tưởng đến phân phối sản phẩm.

1

Discover & Idea Hardening (Phát hiện & Thách thức Ý tưởng)

Mục tiêu: Đánh giá tính khả thi, lập giả thuyết kinh doanh & nghiên cứu bối cảnh dự án.
research-agent analyst-agent

Giai đoạn đầu tiên tập trung vào việc nghiên cứu thị trường, domain kĩ thuật và thách thức các điểm yếu của ý tưởng sản phẩm.

Lệnh chạy đầu tiên
/idea-challenge
/research
Output nhận được

📄 docs/idea-distillate.md (chứa press release & customer FAQ)
📄 research_notes.md (báo cáo nghiên cứu thị trường/technical)

2

Plan (Hoạch Định Yêu Cầu)

Mục tiêu: Chi tiết hóa sản phẩm thành tài liệu đặc tả chức năng rõ ràng.
pm-agent product-agent

Lớp kế hoạch tạo ra Product Requirements Document (PRD) hoàn chỉnh với Success Metrics, Technical Stack và phân chia chức năng rõ ràng.

Lệnh tiếp theo
/create-prd
Output nhận được

📄 docs/PRD.md (Bản PRD chi tiết đạt chuẩn SMART)

3

Solution (Kiến Trúc & Đặc Tả UI UX)

Mục tiêu: Bản vẽ kiến trúc, thiết kế UI UX và bẻ nhỏ Epic & User Story.
architect-agent ux-agent

Chuyển đổi PRD thành kiến trúc kỹ thuật chi tiết (ADR), sơ đồ cơ sở dữ liệu, các Epic & Story sẵn sàng code và các bản đặc tả UI/UX.

Lệnh tiếp theo
/create-architecture
/create-epics-and-stories
Output nhận được

📄 docs/architecture.md (Ghi nhận quyết định kiến trúc - ADR)
📄 docs/epics_list.md & các file story_*.md sẵn sàng để dev-agent nhận việc.

4

Implement (Thực Thi & Viết Code)

Mục tiêu: Code logic tính năng và viết unit test cho từng Story.
dev-agent

Bắt đầu lập kế hoạch sprint và giao việc cho Coding Agent viết code. Các file story được update liên tục trạng thái sang ready, in-progress, review, done.

Lệnh tiếp theo
/sprint-planning
/dev-agent-story
Output nhận được

💻 Source Code sạch chuẩn Clean Architecture.
🧪 Unit Tests tự động hoàn thành (coverage > 80%).

5

Validate (Cổng Đảm Bảo Chất Lượng)

Mục tiêu: Đánh giá chất lượng code, giả lập người dùng, quét bảo mật.
review-agent qa-agent

Chạy kiểm thử tự động, review mã nguồn, phân tích rủi ro tương thích ngược và chạy giả lập người dùng để tìm lỗi logic nghiệp vụ.

Lệnh tiếp theo
/code-review-internal
/simulate-user
Output nhận được

📄 Báo cáo review chi tiết, log quét bảo mật (SAST), và điểm chất lượng Hybrid QA Scorecard.

6

Deliver & Evolve (Bàn Giao & Tiến Hóa Tri Thức)

Mục tiêu: Handoff sản phẩm, cập nhật bối cảnh tri thức, tổng hợp dashboard.
delivery-manager-agent capability-agent

Khi sprint kết thúc, tiến hành họp cải tiến (Retro) để thu hoạch "Knowledge Delta" nạp lại vào Corporate Brain. Biên dịch dashboard để cập nhật trạng thái làm việc.

Lệnh tiếp theo
/retro
iwish gen-dashboard
Output nhận được

📄 docs/KNOWLEDGE_DELTA.md (báo cáo tri thức thu hoạch)
🖥️ Dashboard I-Wish Workspace tương tác được cập nhật mới nhất.

🛠️ Hệ Thống Kỹ Năng Bổ Trợ (Supportive & Foundational Skills)

Bên cạnh luồng chính, I-Wish trang bị hệ thống kĩ năng chuyên biệt làm nhiệm vụ bảo vệ chất lượng phần mềm, quản trị bảo mật và tối ưu trải nghiệm người dùng.

repo-absorption Supportive

Phân tích, hấp thụ và giải cấu trúc các repository bên ngoài từ GitHub, tự động tích hợp tài liệu hướng dẫn và liên kết với Orch routing.

idea-hardening Supportive

Đánh giá nghiêm túc các ý tưởng sản phẩm, phân tích ít nhất 2-3 phương pháp thực hiện kèm theo phân tích trade-offs và giá trị cốt lõi.

security-guardian Foundational

Kiểm soát an ninh vòng ngoài: Quét lỗ hổng, secret leak, rà soát mức độ tin cậy và cấp quyền tối thiểu trước khi code được merge/deploy.

ux-guardian Foundational

Đảm bảo giao diện người dùng đồng bộ tuyệt đối với Master Design System, ngăn chặn tình trạng "lệch giao diện" sau nhiều sprint.

user-simulation-guardian Foundational

Sử dụng tư duy 1st-Principle giả lập hành vi của các Persona người dùng doanh nghiệp khác nhau để test tính mạch lạc của luồng UX.

pivot-guardian Supportive

Người gác cổng vòng lặp: Tự động ngắt agent nếu chỉnh sửa liên tục một tệp quá 3 lần mà không thông qua chạy test hoặc build xác thực.

socratic-review Foundational

Thực hiện hỏi đáp Socratic để rà soát thay đổi cơ sở dữ liệu, kiểm soát tính tương thích ngược và chống drift kiến trúc.

qa-simulator-guardian Supportive

Mô phỏng quy trình kiểm thử toàn diện, tính toán bảng điểm chất lượng Hybrid Scorecard giúp đánh giá độ sẵn sàng ship code.

💡 Các Use Case Cụ Thể & Hướng Dẫn Nhanh

Các kịch bản thực tế khi làm việc với hệ thống I-Wish và các lệnh cần chạy tương ứng.

📂 1. Hấp thụ và bổ sung Skill mới từ GitHub (RAP Protocol)

Khi bạn muốn tích hợp một thư viện, template, hoặc skill bên thứ ba từ một repository công khai trên GitHub vào hệ thống I-Wish.

  • Cách thức hoạt động: Hệ thống kích hoạt quy trình Repo Absorption (RAP). Đầu tiên security-guardian sẽ quét mã độc, sau đó repo-absorption clone và index toàn bộ repo để trích xuất API/Specs, sinh tài liệu integration-guide.md, và đăng ký định tuyến routing cho Orch Agent.
  • Lệnh CLI để thực hiện:
iwish register-module https://github.com/sickn33/antigravity-awesome-skills
🌐 2. Sao chép và lập trình UI từ Website có sẵn (Clone Website)

Khi bạn muốn clone giao diện của một trang web bất kì để chuyển đổi thành components React/HTML code sạch chuẩn trong workspace.

  • Cách thức hoạt động: Gọi workflow /clone-website, agent sử dụng browser tool (Chrome DevTools MCP) để chụp ảnh màn hình, lưu lại cấu trúc DOM và CSS, phân tách các Tokens thiết kế và tự động build các components code.
  • Lệnh slash command trong chat:
/clone-website
🎨 3. Thiết kế & Đồng bộ giao diện (UI UX Pro Max / Stitch)

Khi bạn thiết kế UI và muốn đồng bộ hoá file DESIGN.md của bạn với server Stitch để tạo và tối ưu giao diện.

  • Cách thức hoạt động: Đầu tiên, chạy /create-ux-design và /create-ui-spec để lập spec thiết kế. Sử dụng workflow /sync-stitch-design để đẩy thiết kế lên máy chủ Stitch. Cuối cùng, visual-fidelity-gate sẽ audit tự động xem code frontend có khớp pixel với nguồn thiết kế không.
  • Lệnh slash command:
/create-ui-spec
/sync-stitch-design
📊 4. Phân tích Codebase & Tạo Graph phụ thuộc

Để vẽ được sơ đồ liên kết giữa các file code và các User Story (Feature Graph), giúp Orch Agent hiểu được độ ảnh hưởng khi thay đổi code.

  • Cách thức hoạt động: Công cụ quét toàn bộ source code của bạn, trích xuất import statements, các hàm gọi và ánh xạ chúng vào Feature Graph. Dữ liệu này được hiển thị trực tiếp trên tab "Feature graph" của Dashboard.
  • Lệnh CLI chạy trên terminal:
iwish analyze-codebase
🖥️ 5. Biên dịch & Xem Dashboard I-Wish Workspace

Để tạo ra trang HTML Dashboard tương tác mà bạn đang xem, tổng hợp toàn bộ cây ý tưởng, biểu đồ thực thi và Kanban sprint.

  • Cách thức hoạt động: Lệnh đọc các file metadata trong _iwish-output/ và build thành file HTML tĩnh _iwish-output/dashboard.html. Bạn có thể mở trực tiếp trong trình duyệt để giám sát tiến độ thực tế của AI Agents.
  • Lệnh CLI biên dịch:
iwish gen-dashboard

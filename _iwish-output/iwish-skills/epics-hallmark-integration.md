# Epics & Stories: Hallmark Integration into I-Wish SYSTEM_SKILLS

This document breaks down the Hallmark PRD requirements into Epics and Stories.

---

## Epic 1: UX Guardian & Slop Test Integration

### Story 1.1: Integrate Anti-Slop Rules into UX Guardian
- **Goal:** Merge the key design rules from Hallmark's 69-gate slop test into I-Wish's `UX Guardian` rules.
- **Acceptance Criteria:**
  - AC1: Create `ux-guardian-anti-slop.md` or append rules to `UX Guardian`'s skill rules.
  - AC2: Focus on typography rules, spacing rules, and transition rules.

---

## Epic 2: Visual DNA study Integration

### Story 2.1: Design DNA Extraction Guide
- **Goal:** Add guidelines for Visual DNA Study Mode to the `/clone-website` or `ui-ux` skill directory.
- **Acceptance Criteria:**
  - AC1: Write `design-dna-extraction.md` detailing the 5-step protocol (Surface, Type, Structure, Motion, Rhythm).
  - AC2: Document URL Mode (WebFetch safety + refusal lists) and Image Mode rules.

---

## Epic 3: 8-State Component Previews

### Story 3.1: 8-State Component Preview Generation rule
- **Goal:** Enforce 8-state `.preview.html` generation for UI components.
- **Acceptance Criteria:**
  - AC1: Create a rule file `8-state-preview-rule.md` or similar to guide `dev-agent`.
  - AC2: Define the 8 interactive states structure.

# Product Requirements Document (PRD): Hallmark Integration into I-Wish SYSTEM_SKILLS

## 1. Goal Description
The objective is to integrate the core capabilities of the `hallmark` design skill into I-Wish's system skills and workflows. Specifically, we will merge the **69-Gate Slop Test** rules into `UX Guardian`, integrate the **Visual DNA Study Mode** concept to support visual extraction in `/clone-website`, and implement **8-State Component Previews** validation inside I-Wish's development agent flows. This will dramatically elevate the aesthetic fidelity and code quality of UI pages generated by I-Wish.

---

## 2. Functional Requirements (FR)

### FR1: Ingest UX Anti-Slop Rules
- Merge selected rules from Hallmark's `references/slop-test.md` into I-Wish's `UX Guardian` skill rules.
- Enforce check gates for bad spacing, placeholder text, transition-all statements, color contrast, and lack of active/focus states.

### FR2: Establish Visual DNA Study Mode Support
- Integrate a Design DNA study prompt / checklist under `.agent/skills/ui-ux/` or `clone-website` to enable structural layout, typography, and color extraction before code synthesis.
- Expose the dual URL Mode and Image Mode guidelines.

### FR3: Implement 8-State Component Previews
- Create a template or rule for dev-agent to generate `.preview.html` file rendering the component in all 8 interactive states (Default, Hover, Active, Focus, Disabled, Loading, Selected, Error).
- Set up verification steps.

---

## 3. Non-Functional Requirements (NFR)

### NFR1: Zero Token Bloat
- Avoid copying all 350KB of Hallmark's rulesets directly into the default context. Instead, extract only the essential anti-slop rules and preview guidelines.

### NFR2: Directory Isolation
- Discard all global files, site demos, or vercel configurations from the absorbed Hallmark codebase.

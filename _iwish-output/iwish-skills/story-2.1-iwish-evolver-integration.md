# Story 2.1: Warm-up Overhead Filtering & Time-Budget Enforcement

**Epic:** Epic 2: Time-Budgeted Execution & Metric Gathering
**Story Title:** Warm-up Overhead Filtering & Time-Budget Enforcement
**Goal:** Implement time-guard mechanisms under `.agent/skills/iwish-evolver/scripts/time-budget-filter.py` that enforce strict timeouts on skill runs while filtering out cold-start/warm-up latency (e.g., initial Docker spin-ups, LLM client initialization, embedding model loading) to ensure performance evaluation is fair and consistent.

---

## 🎯 1. Tracer Bullet Identification
**Is this a Vertical Slice?** Yes. It spans:
1. **Runner Guard**: Wrapping process executions with a strict wall-clock time limit.
2. **Warm-up Filter**: Running a dummy baseline/first-flight step, discarding its latency, and averaging subsequent runs.
3. **Interrupt Mechanism**: Exiting early with a clean status code when the budget is exceeded to prevent resource waste.

---

## 📝 2. Acceptance Criteria (AC)
- **AC1:** **Given** a grounding command needs validation, **When** executed under the time budget filter, **Then** it discards the timing of the first run (cold execution) to eliminate warm-up overhead.
- **AC2:** **Given** multiple validation iterations, **When** measured, **Then** it calculates the average latency based strictly on subsequent iterations (warm runs).
- **AC3:** **Given** a command exceeds the allocated `TIME_BUDGET` (e.g., 30s for tool commands, 300s for benchmarks), **When** running, **Then** the process is terminated immediately, raising a clear "TIMEOUT" failure to the evolver.
- **AC4:** **Given** an execution hangs or loops, **When** terminated by the budget guard, **Then** it cleanly releases all subprocess file descriptors, locks, and child PIDs to avoid orphaned processes.

## 💬 5. Socratic Review Synthesis Summary
- **Overhead Elimination**: Discarding the first run prevents LLM JIT loading, filesystem caching, and server handshakes from incorrectly penalizing a skill's throughput score.
- **Process Teardown**: Standard `subprocess.kill()` only kills the parent shell process, leaving running children alive. We must use `os.killpg(os.getpgid(proc.pid), signal.SIGKILL)` to terminate the entire process group.
- **Time Metric**: Using `time.perf_counter()` instead of `time.time()` guarantees high-precision monotonic timing unaffected by system clock adjustments.

# Step IC-02: Press Release

## Goal

Draft the customer-facing promise of the finished product before building it.

## Force Clarity On

- headline
- subheadline
- opening value statement
- problem paragraph
- solution paragraph
- how it works
- getting started

## Coaching Standard

- no vague claims
- no jargon-first language
- no “AI/blockchain/etc” as a substitute for user value
- every paragraph must survive the question: “so what?”
- commercial and non-commercial concepts should be framed differently

## Artifact Update

Write into `idea-challenge-{project}.md` using the main output template.
Append a `<!-- coaching-notes-stage-2 -->` block for rejected framings and differentiation ideas that were considered but not kept.

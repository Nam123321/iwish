# Step U-01: Intake

## Purpose
Capture context, determine phase, and load relevant documents.

## Steps
1. Parse payload (manual text or JSON):
   ```json
   {
     "context": "story-17.1 | prd | architecture | idea",
     "phase": "discovery | planning | architecture | story | dev | review | retro",
     "depth": "full | partial | auto | quick",
     "scope": "macro | micro | bridge | all",
     "target_file": "/path/to/file.md",
     "headless": false
   }
   ```
2. Auto-detect phase if not specified:
   - If `target_file` contains "story-" → `phase=story`
   - If `target_file` contains "prd" → `phase=planning`
   - If `target_file` contains "architecture" → `phase=architecture`
   - If no target_file → ask user
3. Load context documents:
   - `project-context.md` (always)
   - Phase-relevant files (PRD for macro, story + spec for micro)
   - `_iwish-output/unknowns/macro-risks.yaml` (if exists)
   - `_iwish-output/unknowns/unknowns-ledger.yaml` (if exists — for historical patterns)
4. Run `.agent/scripts/uip-filter.py` with payload to get execution plan.

## Exit Criteria
- [ ] Phase identified
- [ ] Context documents loaded
- [ ] Execution plan generated by filter script
- [ ] User confirmed execution plan (unless headless)

Next, proceed to `step-u-02-macro-scan.md` or `step-u-03-micro-scan.md` as determined by routing logic.

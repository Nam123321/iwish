# 🎨 Stitch Design Taste SKILL (Visual Enforcement Gate)

## Purpose

Enforce absolute visual fidelity between the implemented code and the approved Google Stitch screens. This skill prevents the "Schema-Driven Layout" error where developers build UI based on backend data schemas rather than the approved DOM structure.

## Core Directives

1. **Stitch is the Absolute Source of Truth:**
   - The CSS/HTML generated from Stitch is a rigidly enforced visual contract.
   - Do NOT deviate from this contract just because the backend data model (schema) is structured differently.

2. **DOM-Driven Layout:**
   - Your React component hierarchy MUST map to the Stitch DOM hierarchy.
   - Example: If `ConditionType` and `isProgressive` logically sit together in the backend, but the Stitch design separates them visually into different cards, you MUST build the UI with different cards.

## Validation Checklist (The "Taste" Check)

Before handing off any UI story, execute these checks:

- [ ] **Structure Check:** Does the React component tree match the original Stitch HTML structure?
- [ ] **CSS/Tailwind Check:** Did you download and apply the exact CSS tokens generated by Stitch?
- [ ] **Spacing & Typography:** Are the paddings, margins, shadows, and font variants identical to the Stitch mock?
- [ ] **Data Independence:** Is the UI layout decoupled from the backend schema shape? (i.e. not using `map()` blindly on backend objects if that breaks the Stitch grid layout).

## Output Format

If invoking this skill manually or during `/dev-story` compilation checks, output:

> 🛡️ **[STITCH-DESIGN-TASTE] VISUAL GATE CHECK**
> - DOM-Driven Layout: [PASS/FAIL]
> - CSS/Visual Fidelity: [PASS/FAIL]
> 
> *Notes/Corrections:* [List any deviations found and how they were fixed]

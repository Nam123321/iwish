# Figma-to-Code Workflow

## Purpose

Turn approved Figma handoff artifacts into implementation-ready guidance while respecting the project's token system and component architecture.

## Core Responsibilities

1. Extract the approved Figma structure, components, spacing, typography, and states.
2. Map them into:
   - project components
   - design tokens
   - CSS variable/Tailwind usage
3. Produce implementation-facing handoff artifacts.
4. Validate against `visual-fidelity-gate`.

## Expected Inputs

- approved Figma frames/components
- UI spec
- token system
- component library constraints

## Expected Outputs

- component mapping
- token mapping
- implementation guidance
- validation checklist

## Notes

- This is a default skeleton, not yet a Figma-connector-specific execution workflow.

# Playwright-First QA

## Purpose

Provide the default browser QA workflow when the project selects `playwright`.

## Core Responsibilities

1. Decide whether the task is visual QA, interaction testing, or extraction.
2. Run browser-based verification flows with stable reproducible steps.
3. Capture evidence for review, bugfix, or UX alignment workflows.

## Expected Inputs

- selected browser tool profile = `playwright`
- target URL, app state, or scenario
- related story or bug context

## Expected Outputs

- browser evidence
- issue reproduction or validation notes
- suggested follow-up for review or implementation

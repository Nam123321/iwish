# Browser-Use First QA

## Purpose

Provide the default usage-pack skeleton for `browser-use`.

## Core Responsibilities

1. Run lightweight browser validation in the in-app browser context.
2. Capture interactive evidence for design or QA review.
3. Escalate to a stronger browser tool if the task exceeds current browser-use capability.

## Expected Inputs

- selected browser tool profile = `browser-use`
- scenario or page target
- related workflow context

## Expected Outputs

- browser validation notes
- screenshots or evidence references
- recommendation to stay or switch tools

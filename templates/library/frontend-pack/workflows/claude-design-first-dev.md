# Claude-Design-First Workflow

## Purpose

Provide a default usage-pack entrypoint for projects that use Claude Design or similar AI-native design generation as the approved master design source.

## Core Responsibilities

1. Generate or review the approved design concept.
2. Identify which concept/frame/state is the source of truth for implementation.
3. Produce a stable handoff artifact before development starts.
4. Trigger `claude-design-to-code` when implementation mapping is required.

## Expected Outputs

- approved design concept inventory
- selected handoff target
- implementation handoff notes

## Notes

- This is a skeleton pack for orchestration readiness.
- Use together with `visual-fidelity-gate`.

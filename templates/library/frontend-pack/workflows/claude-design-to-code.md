# Claude-Design-to-Code Workflow

## Purpose

Adapt approved Claude Design output into project-constrained implementation guidance.

## Core Responsibilities

1. Extract stable structure from the approved Claude Design output.
2. Map it into:
   - project components
   - token system
   - engineering-ready layout guidance
3. Produce implementation and validation artifacts.
4. Validate with `visual-fidelity-gate`.

## Notes

- This default skeleton is intentionally platform-neutral at the execution detail level.
- Project-specific adapters can deepen it later.

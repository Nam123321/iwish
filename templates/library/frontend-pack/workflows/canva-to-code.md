# Canva-to-Code Workflow

## Purpose

Translate approved Canva output into implementation-ready guidance while preserving the project's design-token and component constraints.

## Core Responsibilities

1. Extract stable structure from the approved Canva artifact.
2. Map it into:
   - component structure
   - design tokens
   - implementation notes
3. Produce handoff material for engineering.
4. Validate with `visual-fidelity-gate`.

## Notes

- This skeleton proves the Open Tool path for a design platform beyond Stitch/Figma.
- Project-specific adapters can deepen it later.

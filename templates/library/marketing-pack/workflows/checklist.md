# MKT Sync Quality Checklist

## Content Quality

- [ ] Feature Announcement has Problem → Solution → Benefit structure
- [ ] Feature Announcement has clear CTA (Call to Action)
- [ ] User Guide has ≥3 screenshots/visuals for illustration
- [ ] User Guide follows step-by-step format with numbered steps
- [ ] Business Use Case has specific persona (name, role, pain point)
- [ ] Business Use Case has measurable ROI or time-saved metric

## Social Media

- [ ] Facebook draft ≤2200 characters
- [ ] LinkedIn draft ≤3000 characters, professional tone
- [ ] Zalo draft ≤500 characters
- [ ] All social drafts have relevant hashtags
- [ ] All social drafts have image/visual attached

## Email Campaign

- [ ] Subject line ≤60 characters and compelling
- [ ] Email body has personalization placeholder {customer_name}
- [ ] Email has clear single CTA
- [ ] Email has unsubscribe note

## Release Notes

- [ ] Uses [NEW] / [IMPROVED] / [FIXED] format
- [ ] Each entry has user-facing impact (not technical jargon)
- [ ] Version number included

## Landing Page Copy

- [ ] Hero headline ≤12 words
- [ ] ≥3 feature highlight blocks
- [ ] Clear primary CTA button text

## Visual Assets

- [ ] All screenshots are clear and readable
- [ ] No sensitive data visible in screenshots (emails, passwords, real customer data)
- [ ] Screenshots show the feature in action, not empty states
- [ ] Banner/visual matches brand colors

## Brand & Voice

- [ ] Vietnamese primary, bilingual where needed
- [ ] Professional but friendly tone
- [ ] No developer jargon in user-facing content
- [ ] Consistent terminology across all materials
